

IP란 인터넷 프로토콜을 말한다.

프로토콜은 약속, 협약 등을 의미하는 단어로 상호간의 원활한 소통/통신 등을 위해 일종의 규칙을 정하는 것이라고 볼 수 있다.

즉, IP는 인터넷에서 상호간의 통신을 위해 지켜야할 협약을 의미한다. 

IP주소는 통신에서 원활한 통신을 위해 집 주소를 나타내는 것이라고 보면 된다.

도메인은 이러한 IP주소를 사람이 쉽게 읽을 수 있게 네이밍한 것이다.

구글을 예시로 들어보면 구글의 IP주소는 `173.194.121.32` 이지만 이러한 IP주소의 접근성이 낮아 쉽게 사용할 수 있게 도메인을 사용하였고 우리가 아는 `google.com` 이 바로 도메인 주소이다.



클라이언트

일반적으로 인터넷이 연결된 장치나 소프트웨어를 말한다.

서버

웹페이지, 사이트, 앱을 저장하는 컴퓨터를 말한다.



클라이언트에서 웹페이지에 접근하려고하면 서버는 클라이언트로 웹 페이지의 사본을 보낸다.



정적 웹페이지와 동적 웹페이지의 차이점



정적 웹페이지와 동적 웹페이지의 차이는 용어 그대로 웹사이트가 변하는지의 차이이다. 우리가 어떤 웹사이트를 방문했을때 서버에서 클라이언트에 웹페이지의 정보를 전달하는데 이러한 정보가 변하지 않는다면 정적 웹페이지라하고 클라이언트의 상호작용에 의해 추가적으로 정보를 전달하거나 수정할 수 있다면 동적 웹페이지라 한다.



- 정적 웹페이지

  - 서버에 저장된 html 문서를 클라이언트에 전송함

  - 모든 사용자가 같은 웹페이지를 봄

  - 상호작용이나 내용이 자주 변하지 않는 페이지에서 사용함 ( 기업 소개 등등 )

  - 서버간 통신이 거의 없기 때문에 속도가 빠르고 간단해서 구축하는 비용이 적게듬

    



- 동적 웹페이지
  - 서버에 저장된 html 문서를 클라이언트에 전송함
  - 클라이언트의 요청에 따라 html에 데이터를 추가하거나 변경함
  - 수정이 용이하여 관리가 쉽고 전체적인 운영 비용이 줄어들 수 있음



HTTP는 무엇이고 요청과 응답메시지 구성은 어떻게 될까



HTTP는 HyperText Transfer Protocol의 약자이다. 프로토콜은 IP에서 배운 개념으로 일종의 규칙이다.

즉, 하이퍼텍스트 전송을 위한 일종의 규칙이 HTTP이고 그 주체는 서버와 클라이언트이다.

HTTP의 기본 포트는 80번이다.

HTTP는 요청과 응답의 구조로 되어있다. 클라이언트가 서버에 요청을 보내면 서버가 응답을 하는 것이다.



### HTTP 요청

시작줄 + 헤더 + 본문으로 구성되어있다.

시작줄 : HTTP메서드 (GET, PUT, POST) 혹은 명사 (HEAD, OPTIONS) + URL + HTTP 버전

```html
GET /index.html HTTP/1.1 <!-- 메서드+URL+HTTP버전의 구조 -->
```

<br>

헤더 : 요청에 대한 추가 정보를 담는 부분이며 HTTP 헤더의 기본 구조를 따른다.

```html 
Content-Length :
Content-Type : 
Host : 
...
```

<br>

본문 : GET, HEAD, DELETE, OPTIONS 같은 요청은 보통 본문이 필요하지 않다.

POST 등 일부 요청은 업데이트를  하기 위해 서버에 데이터를 전송하는데 그럴 때 본문이 필요하다.



### HTTP 응답

상태줄+헤더+본문으로 구성되어있다.

상태줄 : 프로토콜 버전 + 상태 코드 + 상태 텍스트로 구성되어 있다.

```html
HTTP/1.1 404 Not Found.
```

<br>

헤더 : 요청의 헤더와 유사하다.

본문 : 요청의 본문과 유사하다. 일반적으로는 HTML 문서가 전송된다.





프레임워크는 무엇일까

한 전문가는 프레임 워크를 프레임워크란, 소프트웨어의 구체적인 부분에 해당하는 설계와 구현을 재사용이 가능하게끔 일련의 협업화된 형태로 클래스들을 제공하는 것 이라고 정의했다.

간단하게 말하면 무언가를 만들기위한 툴을 제공하는 것이며 종류로는 자바 프레임워크, QRM 프레임워크,  백엔드 프레임워크 등등 다양한 프레임 워크가 존재한다.

라이브러리와 혼동할 수 있는데 유사한 기능을 하지만 결정적인 차이는 제어 흐름이다.

우리가 프레임워크를 사용하는 것은 프레임워크가 동작하기 위해 우리가 코드를 짜넣어서 원활하게 동작하게 만드는 느낌이라면 라이브러리는 우리가 만든 코드가 동작하게 만들기 위해 필요한 라이브러리를 끌어와 사용하는 개념이다.

프레임워크는 주가 프레임워크이고 라이브러리는 주가 사람인 것이다. 이러한 현상을 제어의 역전이라고 한다. 예를 들면 사람과 기계가 있는데 기계를 동작시키기 위해 사람이 일을 한다. 즉, 기계가 사람을 활용하는 현상이 되는 것이 제어의 역전이며, 이러한 기계를 프레임워크라고 하는 것이다. 

반면, 사람이 기계를 활용하여 어떠한 행위를 한다면 이러한 관계가 사람과 라이브러리의 관계라고 볼 수 있다.
